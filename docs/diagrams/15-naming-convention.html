Filename Convention

{SIT}{T}{NN}{AGC}_R_{YYYY}{DOY}{HHMM}_{PERIOD}_{SAMPLING}_{CONTENT}.{TYPE}[.{COMPRESSION}]

Field Width Example Description
Station Block — 9 characters (IGS-compatible site prefix)
SIT 3 chars ROS Site identifier (e.g. ROS = Rosalia, HAI = Hainich)
T 1 char R Receiver role — R = reference (above canopy), A = active (below canopy)
NN 2 digits 01 Receiver number, zero-padded (01–99)
AGC 3 chars TUW Agency / data provider code (e.g. TUW, GFZ, BKG)
Data Source — literal separator
_R_ 3 chars · literal _R_ Data source flag — always _R_ (Receiver); underscore delimiters included
Start Epoch — 11 characters
YYYY 4 digits 2025 Year (UTC)
DOY 3 digits 001 Day of year, zero-padded (001–366)
HHMM 4 digits 0000 Start time in hours and minutes (UTC); 0000 for full-day files
File Parameters — underscore-delimited
PERIOD 3 chars 15M File duration / batch size (e.g. 15M, 01H, 01D)
SAMPLING 3 chars 05S Observation interval (e.g. 05S, 30S, 01S)
CONTENT 2 chars AA Content type code — always AA for canVOD raw observations
TYPE 3–4 chars rnx File format extension: rnx (RINEX), sbf (Septentrio), ubx (u-blox), nmea
COMPRESSION 2–3 chars · optional .gz Compression extension appended after TYPE: gz, zip, zst
